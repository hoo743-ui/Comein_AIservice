# 04. GUI · UX

> **시각 언어 규격(토큰·타이포·색·여백·모션)은 `22_DESIGN_LANGUAGE.md`, 화면 전환·정보 구조(IA)는 `23_USER_JOURNEY.md`가 단일 기준이다.** 이 문서는 그 위에서 **워크스페이스 화면 구성과 UX 원칙**만 다룬다. (구 3분할·글래스·인라인 카드 스펙은 폐기되었다.)

## 무드

**감정적 럭셔리 — "비싸게 보이는" 게 아니라 "힘이 안 드는(effortless)" 느낌.** 대부분 모노크롬 + 브랜드 퍼플 액센트 한 지점. 장식·그라디언트·글래스·큰 그림자를 덜어내고, 여백·타이포 위계·자연스러운 모션으로 깊이를 만든다. 인터페이스는 사라지고 사용자의 사고가 초점이 된다. (근거: `22_DESIGN_LANGUAGE.md`)

## 화면 여정 (4개의 순간)

`/` **Landing**(정체성·철학) → `/experience` **Experience**(시네마틱 리빌 + 로그인) → `/enter` **Enter**(간편 소셜 로그인, "바로 입장") → `/workspace` **Workspace**. 상세 흐름·분기·전환은 `23_USER_JOURNEY.md`.

## 워크스페이스 레이아웃 (슬림 레일 + 단일 캔버스)

```
┌──┬───────────────────────────────────────┐
│레│   단일 캔버스 (뷰가 크로스페이드로 교체)      │
│일│                                        │
│· │   그리팅 · 날씨 · 오늘의 흐름             │
│T │   AI 어시스트 한 줄 (조용한 제안)         │
│C │   좌: 미니 캘린더 / 우: 뷰 콘텐츠          │
│T │                                        │
│N │   [  캡처 바 ───────────────────  ➤ ]   │
│M │                                        │
│P │                                        │
└──┴───────────────────────────────────────┘
 슬림 레일: Today · Calendar · Tasks · Notes · Meetings · People
```

- **중심은 대화가 아니라 워크플로우.** 캡처 바는 여러 입력 방식 중 하나 — 말 한 줄이 일정/할 일/회의/메모로 분류(`classify`)되어 목적지 뷰로 배정된다.
- **페이지 이동이 아니라 한 공간의 재구성.** 뷰 전환은 라우트 이동 없이 캔버스만 **크로스페이드**, 레일·캡처 바는 상시 고정.
- **캡처 결과(receipt)** 는 "무엇이 어디로 정리되었는지"를 조용히 보여준다 — 사용자가 분류하지 않아도 정리된다.
- **AI 어시스트**는 인터페이스를 지배하지 않는다. 한 줄로 맥락(다가오는 일정·미정리 회의·우선순위)을 짚고, 실행 버튼은 최소.
- **문턱(threshold)** — 입장 직후 1회, 문이 열리듯 밝아지는 전환을 재생(`comein:justEntered`). 이후 재방문은 즉시 캔버스.

## 여섯 개의 뷰

| 뷰 | 초점 |
|----|------|
| Today | 오늘의 흐름 — 그리팅·날씨·임박 항목·어시스트 |
| Calendar | 일정과 충돌 |
| Tasks | 할 일과 우선순위 |
| Notes | 생각을 메모로 |
| Meetings | 회의와 요약 |
| People | 연결된 사람 |

## 모션

- 등장·전환은 **확신을 전달**(장식 아님). 등장/안착 `cubic-bezier(0.22,1,0.36,1)`, 디졸브 `cubic-bezier(0.4,0,0.2,1)`, Entrance `700–900ms`, 순차 stagger `80–120ms`로 "정리되는" 느낌.
- `prefers-reduced-motion`을 항상 존중(모션 제거, 결과는 즉시 표시). 상세는 `22_DESIGN_LANGUAGE.md §4`.

## 타이포그래피

- Sans 전부: Inter / SF Pro 계열 그로테스크(`--font-sans`), 한글 `Pretendard Variable`. **세리프는 사용하지 않는다** — 위계는 굵기(200↔600)·자간·리듬으로. 데이터(시각·라벨·좌표)는 Mono.
- 상세 스케일은 `22_DESIGN_LANGUAGE.md §1`.

## 접근성

- 색 대비는 토큰 기반으로 라이트/다크 모두 확보. 포커스는 **기능적 링**(accent ~9%)만 — 유일하게 허용되는 그림자.
- 상태 전달(로딩 스피너·live region 등)은 장식이 아니라 정보로만 사용.
